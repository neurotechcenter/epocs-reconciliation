Extending the documentation for EPOCS

The EPOCS documentation is a work in progress. As at 2014-07-31, only the hardware and programming sections have been written.

The code is a mercurial (hg) repository located at http://bitbucket.org/jezhill/epocs The documentation is managed using the bitbucket wiki system, which creates a wiki that is linked from the main project page. This means that the documentation pages are instantiated as .md files in a second mercurial repository at http://bitbucket.org/jezhill/epocs/wiki You can browse and edit the docs by putting this URL into a browser, or you can use the same URL to clone the repository with mercurial, and edit the .md files with a text editor. Checking files into a local working copy of this repository, committing them, and pushing them back to bitbucket.org, is the best way of adding images and other static files to the documentation.

Making a static copy

The wiki repository also contains a Python script, markdown2static.py, which can be used to convert the .md pages of the wiki repo into .html files which go into the main EPOCS source repo, under the doc directory (from there, they will be bundled into the EPOCS installer next time you compile app/Make-EPOCS-Installer.iss with Inno Setup).

markdown2static.py requires the third-party markdown package. If absent (i.e. if import markdown fails) this can be installed easily from the IPython command-line of a Full Monty distro by typing easy_install markdown. Once this is installed, you can make the static documentation pages by saying:

run markdown2static ../PATH/TO/EPOCS/doc

where ../PATH/TO/EPOCS is the relative path to your working copy of the main EPOCS repo. Its doc subdirectory is designated as the place for static html documentation files. Don't forget to version-control them in the main EPOCS repository: add any new ones and commit any changes before creating a new installer.
