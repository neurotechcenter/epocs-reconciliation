EPOCS Offline Analysis

For developers:

The online EPOCS application can only display an analysis of each GUI tab's most recent run, and only as long as the data are still in memory. Offline Analysis mode allows users to recreate the same "Analysis" window from old files. EPOCS can be run in "Offline Analysis" mode by one of the following methods:

  • Launch via a shortcut to app/gui-bin/epocs-offline.exe
  • Run from source by launching via a shortcut to app/gui/offline.bat
  • Run from source from an IPython prompt by typing run epocs --offline

Note that the "run from source" options require installation of a Python distribution with certain third-party packages, as detailed on the EPOCS GUI page.

EPOCS Offline Analysis allows multiple files to be analysed at once (shift-click or ctrl-click on them to select multiple files in the dialog). As at 2014-07-31 it has the following properties and limitations:

  • No Voluntary Contraction analysis is performed, and no usable information will be read from files ending -VC.dat. The application displays the timing, sequence and distribution of time-locked responses, as in EPOCS' online analysis of Control Trials or Training Trials. This is suitable for files ending -ST.dat, -RC.dat, -CT.dat or -TT.dat.

  • It takes a few seconds to read each file.

  • The application is a console application. Log messages go to the black console window, and not to a file.

  • Errors are also displayed on the console window, which may then close a split second later. Run from a Command Prompt if there are error messages you need to read.

  • The log also automatically goes to the clipboard, and can be pasted elsewhere (a notepad file, or the "Log" tab of an online EPOCS session) using ctrl-v (NB: this seems to work fine from the compiled epocs-offline.exe, but when you're running from source there is one quirk: you must paste before the console application quits to avoid losing the information).
